Rasynergy API Documentation

Company: Rasynergy Digital Card Solution and Printing Company

Submission Date: June 2025

Purpose: Security Assessment by INSA

1. Introduction

Rasynergy is a digital solution provider based in Addis Ababa, Ethiopia, offering digital ID card and student service management. This documentation presents the API system powering the platform available at https://rasynergy.et.

2. Objective

The objective of this document is to provide INSA with a complete overview of our API endpoints, authentication methods, and data exchange patterns for the purpose of conducting a thorough cybersecurity assessment and ensuring compliance with national standards.

3. API Authentication

POST /api/login

Description: Authenticate a user and receive a JWT token. 

{
  "username": "admin",
  "password": "password123"
}
{
  "token": "eyJhbGciOiJI..."
}

Use the token in headers:

Authorization: Bearer <token>

4. API Endpoints Overview

Schools

GET /api/schools – List all schools

POST /api/schools – Create a new school

Students

GET /api/students – List or filter students

POST /api/students – Add a new student

PUT /api/students – Update a student record

Products & Orders

GET /api/products – Get all products

POST /api/products – Create a product purchase

GET /api/orders – Get orders by reference number or student

Payments

POST /api/ras-verify-payment – Start payment process

POST /api/ras-payment-webhook – Payment callback endpoint

5. Authorization and Access Control

All routes are secured using a JWT token passed in the request header. Access is restricted based on user roles (admin, school staff, superadmin).

6. Third-Party Integrations

  • Telebirr (mobile wallet payment integration)
  • SantimPay (payment aggregator)

7. Compliance

All user data is handled securely following local data protection policies. We also ensure:

  • End-to-end encryption during authentication and payment
  • Role-based access control
  • API input validation and request throttling
